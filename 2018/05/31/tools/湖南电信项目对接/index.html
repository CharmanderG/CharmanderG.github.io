湖南电信项目对接

发表于 2018-05-31 | 分类于 Tool

电信对接

程序入口在Plug.c文件

主要逻辑是循环2分钟发一次包

修改点

  • PlugSdk.c

REPO指令

repo init -u ssh://xxxxxxxx@openai-corp.rokid.com:29418/amlogic_a113_audio/manifest -m xmly.xml –repo-url=ssh://xxxxxxxx@openai-corp.rokid.com:29418/tools/repo –no-repo-verify

代码集成

  • 沿用四川电信使用的仓库 open-platform/client/telecom_ipdevc 新建一个湖南电信的分支(eg:hunan),仓库对应的地址是:https://openai-corp.rokid.com/#/admin/projects/open-platform/client/telecom_ipdevc
  • 提交湖南电信的对应代码到湖南电信分支hunan
  • 在manifest里面的xmly.xml中可以修改revision为湖南电信的分支hunan 同步代码并编译

编译

  • source环境:

  • 编译整个镜像:make -j40

  • 编译单个模块:make module-name-rebuild(eg:make telecom_ipdevc-rebuild)
  • 编译生成的产物以及安装路径:(eg:/usr/bin/install -D -m 0755 /home/hs/linux/output/lua_mini_telecom_hunan_a113/build/telecom_ipdevc/telecom_ipdevc /home/hs/linux/output/lua_mini_telecom_hunan_a113/target/usr/bin/telecom_ipdevc 表示生成的可执行文件是/home/hs/linux/output/lua_mini_telecom_hunan_a113/build/telecom_ipdevc/telecom_ipdevc 并且需要安装到usr/bin/目录下)

    gerrit

  • gerrit提交代码 git push origin-name HEAD:refs/for/branch-name(eg:git push openlinux HEAD:refs/for/master)然后还需要登录gerrit做codereview并最终submit这次提交才算完成

查看日志

  • journalctl -u telecom_ipdevc -f(缺陷 日志非实时)
  • systemctl stop telecom_ipdevc; telecom_ipdevc这个是直接运行程序 可以直接看到日志
